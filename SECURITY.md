# Security Policy

## Supported Versions

This project is small and currently supports only the latest state of the `main` branch.

## Reporting a Vulnerability

Do not open public issues for suspected security vulnerabilities.

Instead, report the issue privately to the project maintainer with:

- A clear description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested remediation or patch details

Until a dedicated security contact is configured, report issues through a private maintainer contact method listed on the repository profile or through private vulnerability reporting if the hosting platform supports it.

## Response Process

- Reports will be reviewed as quickly as practical.
- Valid issues will be reproduced, triaged, and fixed based on severity and scope.
- If the report is accepted, a fix will be prepared and released before public disclosure when feasible.
- Reporters acting in good faith will be credited if they want acknowledgment.

## Disclosure Expectations

Please allow reasonable time to investigate and remediate the issue before disclosing it publicly.
